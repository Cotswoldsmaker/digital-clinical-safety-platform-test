How does this all work?

The Clinical Safety Hazard Documentation (CSHD) app has been designed to improved and simplifier the process of documentating the safe design and implementation of digital health systems.

CSHD uses a Django (python based) web app (the one you are using now) to collect details to initialise your clinical safety documents. You can then upload the initialised documents to Github and the documents can be viewed there. Once setup, you will be able to view the documents in an mkdoc server. These docuements are then available as a static site on Github.

What you see on screen will be what is displayed on Github. If you need to update placeholders or the prose of the safety documents that can be done within the Django part of CSHD.
